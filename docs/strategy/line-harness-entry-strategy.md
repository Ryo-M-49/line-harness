## 前提条件

- 個人開発者（エンジニア）
- Cloudflare Pages/Workersのデプロイ経験あり、セットアップは問題なし
- Claude Codeを日常的に使いこなしている
- 初期の資金投下は最小限にしたい

## 1. 何を売るか — サービス設計

### 1.1 既存プレイヤーの弱点（＝攻めるべきポイント）

現在のLINE構築代行市場には明確な構造的弱点がある:

| 既存の問題 | 発生理由 | 自分の武器 |
|-----------|---------|-----------|
| **「構築して終わり」が大半** | 運用まで面倒を見ると人件費が合わない | AIで運用を自動化できるから月額サポートが低コストで提供可能 |
| **ブラックボックス化** | Lステップの設定が複雑で引き継ぎが困難 | LINE HarnessはOSS+API公開。設定がコードで管理され透明 |
| **追加費用が不透明** | シナリオ変更のたびに見積もり | AIで変更対応がほぼゼロコストなので定額で吸収できる |
| **ツール月額が高い** | Lステップ月額2万〜3.3万円がクライアント負担 | LINE Harnessは0円。ツールコスト不要 |
| **BAN対策がない** | 既存ツールにBAN検知機能なし | ステルス配信+自動ヘルスチェック+自動移行が標準搭載 |
| **1人で数アカウントが限界** | 手作業なので労働集約型 | マルチアカウント+AI自動運用で数十アカウント管理可能 |

### 1.2 サービスの核 — 「AI LINE運用」

**従来のLINE構築代行との根本的な違いを打ち出す。**

従来: 「人が手作業でLステップを設定する」→ 構築に数十万、変更のたびに追加費用
提案: 「AIが構築も運用も回す」→ 低コスト・高速・継続改善

**サービス名案**: 「LINE自動化パートナー」「AI LINE運用」等（要検討）

### 1.3 サービスメニュー（3層構成）

#### Tier 1: LINE構築パッケージ（単発）
**対象**: まだLINE公式アカウントを活用できていない事業者
**価格**: 5万〜15万円（市場相場の1/3〜1/10）

| 含まれるもの | 詳細 |
|-------------|------|
| LINE公式アカウント開設・初期設定 | プロフィール、あいさつメッセージ |
| リッチメニュー作成・設定 | 業種別テンプレートベース |
| ステップ配信シナリオ設計・設定 | 5〜10通のシナリオ |
| セグメント・タグ設計 | 業種に最適化した分類設計 |
| 自動応答設定 | よくある質問への自動回答 |
| 納品後30日サポート | チャットでの質問対応 |

**原価**: LINE Harnessのセットアップ+Claude Codeでのシナリオ生成 → 実作業2〜4時間
**利益率**: 極めて高い（時給換算1万〜5万円）

#### Tier 2: 月額運用プラン（サブスク・本命）
**対象**: 構築済みだが運用リソースがない事業者
**価格**: 月額2万〜5万円（市場相場の1/3〜1/5）

| 含まれるもの | 詳細 |
|-------------|------|
| 月4〜8回の配信代行 | コンテンツ企画+作成+配信 |
| セグメント配信の最適化 | 開封率・クリック率ベースのチューニング |
| リッチメニューの更新 | 季節・キャンペーン対応 |
| 月次レポート | 友だち数推移、配信成果、改善提案 |
| シナリオ改善 | A/Bテスト、分岐最適化 |
| BAN監視 | ヘルスチェック+リスク時アラート |

**原価**: Claude Codeで大半を自動化 → 1アカウントあたり月2〜3時間の管理作業
**スケーラビリティ**: 10アカウントで月20〜50万円、30アカウントで月60〜150万円

#### Tier 3: 戦略コンサル+運用（高単価）
**対象**: LINEで売上を本気で伸ばしたい中小企業
**価格**: 月額10万〜20万円

| 含まれるもの | 詳細 |
|-------------|------|
| Tier 2の全内容 | — |
| 月1回の戦略ミーティング（30分〜1時間） | 施策の方向性・KPI確認 |
| スコアリング設計・運用 | 見込み度に応じた自動振り分け |
| CV計測・アフィリエイト連携 | 流入元分析、ROAS追跡 |
| Webhook連携（Stripe/予約システム等） | 決済・予約と連動した自動配信 |
| BAN時の自動移行対応 | UUID統合による友だち引き継ぎ |

## 2. 誰に売るか — ターゲット選定

### 2.1 最有力ターゲット: 美容室・サロン

**理由:**
- LINE予約との相性が抜群（Web予約の7割がLINE経由になった事例あり）
- リピーター獲得にLINEが直結する（クーポン、ショップカード）
- 店舗数が多い（全国25万店超）→ 市場が大きい
- IT知識が乏しい経営者が多い → 代行ニーズが高い
- 客単価が読みやすく、LINE施策のROIを示しやすい

**刺さるメッセージ**: 「ホットペッパーの手数料、LINEで減らせます」

### 2.2 第2ターゲット: EC・通販

**理由:**
- LINE経由の売上2倍の事例あり
- セグメント配信+ステップ配信の効果が数値で見えやすい
- Stripe/Shopify連携でWebhook自動化が刺さる

### 2.3 第3ターゲット: コーチ・コンサル・情報発信者

**理由:**
- 現在Lステップの最大ユーザー層 → 乗り換え需要
- ツール月額2万〜3.3万円を0円にできるコスト訴求
- ステップ配信+スコアリングのニーズが高い
- ただしBAN対策に最も需要のある層（規約リスク高）でもある

### 2.4 ターゲット優先順位

| 順位 | 業種 | 理由 |
|------|------|------|
| 1 | 美容室・サロン | 市場大、ニーズ明確、成果を見せやすい |
| 2 | 整体院・クリニック | 美容室と類似構造、リピーター重視 |
| 3 | EC・通販 | 数値で成果が見える、Webhook連携が差別化 |
| 4 | コーチ・コンサル | Lステップからの乗り換え需要 |
| 5 | 飲食店 | 数は多いが予算が少ない。実績作りには良い |

## 3. どう売るか — 参入ロードマップ

### Phase 0: 準備（1週間）

**目標: 売り物を作る**

- [ ] LINE Harnessをデプロイし、自分のテスト用LINE公式アカウントで一通り操作する
- [ ] Claude Codeから自然言語で主要操作（ステップ配信設定、セグメント作成、リッチメニュー切替）を実行し、所要時間を計測する
- [ ] 業種別テンプレートを3つ作成（美容室、整体院、EC）
  - あいさつメッセージ
  - リッチメニュー
  - 5通ステップ配信シナリオ
  - タグ・セグメント設計
- [ ] デモ用LINE公式アカウントを1つ作り、実際に動く状態にする（ポートフォリオ用）
- [ ] サービス説明資料を作成（料金表・事例・差別化ポイント）

### Phase 1: 実績作り — ココナラ/クラウドワークス（1〜2ヶ月）

**目標: 5件の実績+レビューを獲得する**

#### ココナラ出品戦略

**出品1: 格安構築パッケージ（実績作り価格）**
- 価格: 29,800円（先着5名限定）→ 通常価格 59,800円
- 内容: Tier 1 の構築パッケージ
- 差別化文言:
  - 「ツール月額0円で運用できるLINE自動化」
  - 「エンジニアが構築するから追加費用なし」
  - 「BAN対策機能付き（業界初）」
- ポイント: 最初の5件は赤字覚悟で★5レビューを全力で取りに行く

**出品2: スポットコンサル**
- 価格: 5,000円/30分
- 内容: 「あなたのLINE公式アカウント、無料診断します」
- 目的: 見込み客との接点を作り、構築案件にアップセル

#### クラウドワークスの使い方
- LINE構築案件に提案する（実績作り期間は相場より安く提案）
- 「Lステップではなく、OSSツールでツール月額0円にできます」を武器に差別化
- ココナラより単価が低くなりがちなので、Phase 1の実績作りに割り切る

#### この期間のKPI
- 5件以上の納品完了
- ★4.5以上の平均レビュー
- 1件以上のTier 2（月額運用）への転換

### Phase 2: 直営業 — 地域密着（2〜4ヶ月目）

**目標: 月額運用クライアントを5〜10件獲得する**

#### 営業チャネル

**① Googleマップ営業（美容室・サロン特化）**
- 地元エリアのGoogleマップで美容室・サロンを検索
- LINE公式アカウントを持っていない or 活用できていない店舗をリストアップ
- 「御社のLINE活用、無料で改善提案します」でDM or 直接訪問
- 無料改善提案 → 構築案件 → 月額運用へ
- 1日10件アプローチ → 月200件 → 成約率3%で6件/月

**② Instagram DM営業**
- 美容室・サロンのInstagramアカウントに直接DM
- 「LINEの友だち追加を自動化して、ホットペッパーの手数料を減らしませんか？」
- Instagramとの連携提案も含めて価値を出す

**③ X（Twitter）での発信**
- 「LINE運用の自動化」「LINE構築の裏側」をテーマに発信
- Claude CodeでLINE設定を自然言語で操作するデモ動画を投稿（バズ狙い）
- Building in Publicスタイルで技術的な差別化をアピール

**④ 紹介・口コミ**
- Phase 1の納品先に「お知り合いの店舗にもご紹介いただけませんか？」
- 紹介報酬（初月無料等）を設定

#### 提案テンプレート（美容室向け）

```
件名: LINEで予約管理を自動化しませんか？（無料診断）

〇〇様

突然のご連絡失礼します。
LINE公式アカウントの自動化を専門にしている〇〇と申します。

御店舗のInstagram/Googleマップを拝見し、
LINEを活用すれば以下のことが実現できると考えご連絡しました:

✅ 予約受付をLINEで自動化（電話対応を削減）
✅ 来店後の自動フォローメッセージでリピート率UP
✅ クーポン配信で空き時間を埋める
✅ ホットペッパーの手数料を削減

現在、無料でLINE活用の改善提案を行っています。
15分ほどお時間をいただければ、具体的な提案をお出しします。

ご興味があればお気軽にご返信ください。
```

### Phase 3: スケール — 仕組み化（4ヶ月目〜）

**目標: 月額運用クライアント20〜30件、月売上50万〜100万円**

#### やること

**① 業種別テンプレートの拡充**
- 美容室、整体院、飲食店、EC、コーチ業…業種ごとに構築テンプレートを用意
- Claude Codeのプロンプト+LINE Harness APIで「業種名を入れたら5分で構築完了」を実現

**② 運用の自動化パイプライン構築**
- Claude Code + cronで定期的に各アカウントのKPIを自動取得
- 配信コンテンツの下書きをAIで自動生成 → 自分がレビュー → 配信
- BAN監視は完全自動化（アラートのみ確認）
- 月次レポートの自動生成

**③ ストック収入の積み上げ**
- Tier 2（月額2万〜5万）を主軸に、月額課金アカウントを増やす
- 20アカウント × 月額3万円 = 月60万円
- 30アカウント × 月額3万円 = 月90万円
- AI自動化により1アカウントあたりの管理時間は月2〜3時間
- 30アカウントでも月60〜90時間 → 1人で十分回せる

**④ コンテンツマーケティング**
- note/Zenn/Qiitaで「LINE運用をAIで自動化した話」を発信
- 「Lステップ vs OSS（LINE Harness）」比較記事で検索流入を狙う
- saas-hikaku.netにLINEツール比較カテゴリを追加（自然な集客導線）

### Phase 4: 事業拡大（6ヶ月目〜）

**目標: 月売上100万円超、仕組みの横展開**

#### 選択肢

**A. クライアント数を増やす（50〜100アカウント）**
- Claude Codeの自動化をさらに深化させ、管理効率を上げる
- 一部業務を外注化（配信コンテンツの企画・デザイン）

**B. 構築テンプレートのパッケージ販売**
- 「美容室向けLINE自動化キット」をnote/Gumroadで販売（3万〜5万円）
- LINE Harnessのデプロイマニュアル+業種別設定ファイル+シナリオテンプレート
- スケーラブルだがサポートコストに注意

**C. マネージドホスティングSaaS化**
- LINE Harnessを1クリックでデプロイできるサービスを構築
- 月額5,000〜1万円/アカウント
- 「自分のCloudflareにデプロイしなくていい」だけで価値が成立
- ただし開発コスト・サポートコストが発生する

### 従来の構築代行との比較表（営業資料用）

| 項目 | 従来のLINE構築代行 | 自分のサービス |
|------|-------------------|---------------|

## 6. リスクと対策

| リスク | 深刻度 | 対策 |
|--------|--------|------|
| LINE Harnessの開発停止 | 中 | MITなのでフォーク可能。技術スタックが標準的（CF Workers+D1）なのでメンテ可能 |
| LINE APIの規約変更 | 中 | Messaging API自体は安定。BAN基準変更時はステルスモードのパラメータ調整で対応 |
| 既存ツール（Lステップ等）のAI統合 | 中 | API非公開のまま表面的なAI機能追加にとどまる可能性が高い。根本的なアーキテクチャの差は埋まらない |
| 価格競争（ココナラ等） | 高 | Phase 1の実績作り以降は、ココナラから卒業して直営業に切り替える。月額サブスクで安定収入にシフト |
| スケール時の品質低下 | 中 | 業種別テンプレートと自動化パイプラインで品質を標準化。手動対応が必要な部分を明確化 |
| クライアントの「自分でやりたい」ニーズ | 低 | Tier 1で構築だけ請け負い、運用マニュアルを納品する選択肢も用意 |
